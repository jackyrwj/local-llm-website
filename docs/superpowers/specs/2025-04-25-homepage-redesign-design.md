# 首页改版设计文档 — AI 部署手记

**日期**: 2026-04-25  
**主题**: 首页布局改版 — 突出工具特点，吸引流量，符合 Google AdSense 审核标准  
**状态**: 待用户审核

---

## 1. 设计目标

| 目标 | 说明 |
|------|------|
| **突出工具** | 首页首屏以 5 个核心工具为绝对主角，用户 3 秒内理解网站价值 |
| **吸引流量** | 清晰的价值主张 + SEO 友好的内容结构 + 问题导向的文案 |
| **AdSense 合规** | 原创内容、必要法律页面、不遮挡内容的广告位、良好的用户体验 |
| **降低杂乱感** | 精简元素，明确信息层级，每个区域有单一目的 |

## 2. 信息架构

```
首页 (index.html)
├── Header (固定导航)
│   ├── Logo + 站点名称
│   └── 导航: 首页 | 工具箱 | 文章 | 关于 | 联系
├── Hero Section
│   ├── 主标题: "本地大模型部署工具箱"
│   ├── 副标题: 一句话价值主张
│   └── CTA 按钮: "开始使用" | "查看教程"
├── Stage Filter (阶段筛选标签)
│   └── 全部工具 | 准备阶段 | 启动阶段 | 运行阶段
├── Tools Grid (5 个工具卡片)
│   ├── 上行: 显存估算器 | 成本计算器 | vLLM 命令生成器
│   └── 下行: 报错诊断器 | 部署选型助手
├── AdSense Slot 1 (内容流广告位)
├── Latest Articles (最新文章)
│   └── 3 篇文章卡片 (标题 + 缩略图 + 分类 + 日期)
├── Trust Bar (信任背书)
│   └── 5 工具 · 20+ 文章 · 0 后端依赖 · 浏览器本地计算
└── Footer
    ├── 快速链接 | 法律信息
    └── 版权声明
```

## 3. 各区域详细设计

### 3.1 Header (固定导航)

- **背景**: 白色 + `backdrop-filter: blur(12px)`，滚动时增加阴影
- **高度**: 64px
- **Logo**: 方块图标 + "AI 部署手记" 文字
- **导航项**: 首页、工具箱、文章、关于、联系
- **移动端**: 汉堡菜单，展开全屏导航

### 3.2 Hero Section

- **背景**: 深色渐变 (`#0F172A` → `#1E293B`)，无图片（加载快）
- **内容居中**:
  - 眉标 (Eyebrow): "🔧 本地大模型部署第一站"
  - 主标题 (H1): "本地大模型部署工具箱"
  - 副标题: "从选型到上线，5 个工具帮你搞定 vLLM、GPUStack、Qwen 部署"
  - CTA 按钮组:
    - 主按钮: "开始使用" → 滚动到工具区域
    - 次按钮: "查看教程" → 文章页

### 3.3 Stage Filter (阶段筛选标签)

- **位置**: Hero 下方，工具网格上方
- **样式**: 水平排列的 pill 形状标签
- **选项**:
  - `全部工具` (默认激活，蓝色背景)
  - `准备阶段` (浅蓝标签)
  - `启动阶段` (浅橙标签)
  - `运行阶段` (浅绿标签)
- **交互**: 点击过滤工具卡片（平滑动画），URL 同步 `?stage=prep`

### 3.4 Tools Grid (工具卡片网格)

**布局**: 上行 3 个 + 下行 2 个居中 (3+2)

**单个卡片结构**:
```
┌─────────────────────┐
│ [准备]              │  ← 右上角阶段标签
│                     │
│   [图标]            │
│   显存估算器         │
│   输入模型参数，      │
│   估算 GPU 显存      │
│                     │
│   💡 最常用          │  ← 底部提示标签
└─────────────────────┘
```

**5 个工具详情**:

| 工具 | 阶段 | 标题 | 描述 | 标签 |
|------|------|------|------|------|
| 显存估算器 | 准备 | 大模型显存估算器 | 输入模型参数，估算 GPU 显存是否够用 | 💡 最常用 |
| 成本计算器 | 准备 | 本地 vs API 成本计算器 | 对比自建 GPU 与云 API 的月成本 | 📊 省钱利器 |
| vLLM 命令生成器 | 启动 | vLLM 启动命令生成器 | 按模型和显卡生成可复制的启动命令 | ⚡ 一键生成 |
| 报错诊断器 | 启动 | 部署报错诊断器 | 粘贴报错文本，快速定位问题原因 | 🔧 救急必备 |
| 部署选型助手 | 运行 | 模型部署选型助手 | 根据场景推荐 Ollama / vLLM / API | 🎯 选型指南 |

**卡片样式**:
- 白色背景，圆角 8px，细边框
- Hover: 边框变蓝 + 轻微上浮阴影
- 阶段标签: 右上角小 pill，颜色区分阶段

### 3.5 AdSense Slot 1 (广告位)

- **位置**: 工具网格和文章区域之间
- **样式**: 与内容流自然融合，不突兀
- **尺寸**: 自适应 (responsive)
- **注意**: 不遮挡工具卡片，不与内容混淆

### 3.6 Latest Articles (最新文章)

- **标题**: "最新实战文章"
- **布局**: 3 篇横排（桌面），垂直堆叠（移动端）
- **单篇文章卡片**:
  - 左侧缩略图 (120x80)
  - 右侧: 分类标签 + 标题 + 日期/阅读时长
- **分类标签颜色**: 教程(蓝)、踩坑(橙)、对比(紫)

**展示文章** (3 篇):
1. vLLM + Qwen3 完整部署指南：从零到 API 服务 (教程)
2. Qwen 模型上下文爆满问题解决方案 (踩坑)
3. 本地模型 vs 云端模型：成本全面对比 (对比)

### 3.7 Trust Bar (信任背书)

- **位置**: 文章区域下方，Footer 上方
- **内容**: 4 个数据点横向排列
  - 5 个工具
  - 20+ 篇文章
  - 0 后端依赖
  - 浏览器本地计算
- **样式**: 浅灰背景，大号数字 + 小号文字

### 3.8 Footer

- **三列布局**: 关于站点 | 快速链接 | 法律信息
- **底部栏**: 版权 + "本地 AI 实战记录"

## 4. 响应式断点

| 断点 | 布局调整 |
|------|----------|

## 7. 设计决策记录

1. **去掉搜索框**: 用户明确选择。搜索框会分散工具卡片的注意力，且当前工具数量少（5 个），直接浏览更高效。
2. **3+2 网格**: 视觉平衡，上行满宽、下行居中留白，引导视线自然下移。
3. **阶段标签可点击**: 帮助用户按部署阶段快速定位工具，提升转化率。
4. **深色 Hero**: 与下方白色内容区形成强对比，建立清晰的信息层级。
5. **文章仅展示 3 篇**: 工具为主、文章为辅的策略，避免信息过载。
